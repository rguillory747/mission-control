Module 1: Introduction to Mission Control
What is Mission Control?
Imagine you have five AI agents running simultaneously. One is monitoring your emails, another is posting to social media, a third is analyzing data, and two more are handling customer support tickets. Everything seems fine... until one of them stops working and you don't notice for three hours.

Mission Control solves this problem.

Mission Control is a real-time dashboard that gives you complete visibility into your agent fleet. At a glance, you can see:

Which agents are running and which are down

What each agent is currently doing

When an agent last checked in

What tasks are assigned and their status

A live feed of all agent activity

Think of it like air traffic control, but for your AI agents. You're the operator, and Mission Control ensures nothing crashes without you knowing.

Why You Need This
The Problem with Invisible Agents
When you run AI agents in the background (especially autonomous ones), you face several challenges:

1. Silent Failures

Your agent crashes, but you don't know until someone complains or you manually check logs. By then, you've lost hours of productivity.

2. No Visibility

You have no idea what your agents are actually doing. Are they working? Stuck? Waiting for something?

3. Coordination Chaos

With multiple agents, how do you know who's doing what? How do you prevent duplicate work or missed tasks?

4. Manual Monitoring

Checking logs, SSHing into servers, running status commands – it's tedious and doesn't scale.

5. Debugging Nightmares

When something goes wrong, you're digging through text files trying to reconstruct what happened.

The Solution: Mission Control
Mission Control gives you a single source of truth for your entire agent infrastructure:

Heartbeat monitoring ensures you know within seconds when an agent goes down

Real-time activity feeds show you exactly what's happening right now

Task assignment lets you coordinate work across multiple agents

Visual dashboard provides instant status checks without reading logs

Historical data helps you debug issues and understand agent behavior

Step 7: Set Up Convex Provider
We need to connect our Next.js app to Convex. Create a new file:

"use client";

import { ReactNode } from "react";
import { ConvexProvider, ConvexReactClient } from "convex/react";

const convex = new ConvexReactClient(process.env.NEXT_PUBLIC_CONVEX_URL!);

export function ConvexClientProvider({ children }: { children: ReactNode }) {
  return <ConvexProvider client={convex}>{children}</ConvexProvider>;

How Convex Works
Before we write code, let's understand what makes Convex special.

Traditional Backend vs Convex
Traditional approach:

Set up a database server (PostgreSQL, MongoDB, etc.)

Write API endpoints (Express, FastAPI, etc.)

Handle WebSocket connections for real-time

Write database queries

Handle authentication, caching, error handling

Deploy and manage infrastructure

With Convex:

Define your schema

Write functions

That's it

Key Concepts
1. Schema = Your Database Structure

You define tables and their fields in TypeScript:

agents: defineTable({
  name: v.string(),
  status: v.string(),
})
This creates an agents table with name and status fields.

2. Queries = Read Data

Queries fetch data and automatically re-run when data changes:

export const getAllAgents = query(async (ctx) => {
  return await ctx.db.query("agents").collect();
});
Call this from React and it subscribes to live updates.

3. Mutations = Write Data

Mutations create, update, or delete data:

export const createAgent = mutation(async (ctx, { name }) => {
  return await ctx.db.insert("agents", { name, status: "offline" });
});
4. Everything is Type-Safe

TypeScript knows exactly what fields exist and their types. Your editor will autocomplete and catch errors.

Step 1: Create the Schema File
Create a new file in the convex/ directory:

touch convex/schema.ts
This file defines your entire database structure.

File: `convex/schema.ts`

import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";

export default defineSchema({
  agents: defineTable({
    name: v.string(),
    status: v.union(v.literal("online"), v.literal("offline"), v.literal("error")),
    lastHeartbeat: v.number(),
    metadata: v.optional(v.object({
      version: v.optional(v.string()),
      platform: v.optional(v.string()),
    })),
  })
  .index("by_status", ["status"])
  .index("by_lastHeartbeat", ["lastHeartbeat"]),
});
Let's break this down:

Understanding the Schema
Import statements:

import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";
defineSchema: Creates the database schema

defineTable: Defines a single table

v: Validator for field types

The agents table:

agents: defineTable({
  name: v.string(),
  status: v.union(v.literal("online"), v.literal("offline"), v.literal("error")),
  lastHeartbeat: v.number(),
  metadata: v.optional(v.object({
    version: v.optional(v.string()),
    platform: v.optional(v.string()),
  })),
})
Field explanations:

`name: v.string()`

  - Agent's name (e.g., "email-monitor", "twitter-bot")

  - Required field

  - Type: string

`status: v.union(...)`

  - Current agent status

  - Must be exactly "online", "offline", or "error"

  - v.union = one of these options

  - v.literal = exact string match

  - Type-safe! Can't accidentally set status to "active" or "dead"

`lastHeartbeat: v.number()`

  - Timestamp of last heartbeat (milliseconds since epoch)

  - We'll use Date.now() to set this

  - Type: number

`metadata: v.optional(v.object(...))`

  - Optional additional information

  - Can include version, platform, etc.

  - v.optional = field can be omitted

  - Nested object with optional fields

Indexes:

.index("by_status", ["status"])
.index("by_lastHeartbeat", ["lastHeartbeat"])

function formatTimestamp(timestamp: number): string {
  const seconds = Math.floor((Date.now() - timestamp) / 1000);
  
  if (seconds < 60) return `${seconds}s ago`;
  if (seconds < 3600) return `${Math.floor(seconds / 60)}m ago`;
  if (seconds < 86400) return `${Math.floor(seconds / 3600)}h ago`;
  return `${Math.floor(seconds / 86400)}d ago`;
}

export const deleteOldActivities = mutation({
  args: {
    olderThanDays: v.number(),
  },
  handler: async (ctx, args) => {
    const cutoffTime = Date.now() - args.olderThanDays * 24 * 60 * 60 * 1000;
    
    const oldActivities = await ctx.db
      .query("activities")
      .withIndex("by_timestamp")
      .filter((q) => q.lt(q.field("timestamp"), cutoffTime))

  },

Understanding Heartbeat Systems
What is a Heartbeat?
A heartbeat is a periodic signal that says "I'm still alive." It's used in distributed systems to detect failures without constant monitoring.

Analogy: Like a fitness tracker checking your pulse. If it stops detecting a heartbeat, something's wrong.

How It Works
1. Agent sends heartbeat:

Every 30 seconds:
  POST /api/heartbeat
  { agentId: "abc123", status: "online" }
2. Server updates timestamp:

Update agents table:
  lastHeartbeat = Date.now()
  status = "online"
3. Server checks for missing heartbeats:

Every 60 seconds:
  Find agents where:
    lastHeartbeat < (now - 2 minutes)
  Set their status to "offline"
Why Heartbeats?
Without heartbeats:

Agent crashes → You never know

Network fails → Silent failure

Process hangs → Looks fine, does nothing

With heartbeats:

Agent crashes → Heartbeat stops → Status = offline

Network fails → Heartbeat stops → Alert

Process hangs → Heartbeat continues → Different issue to debug

Step 1: Create the Heartbeat Mutation
Let's add a heartbeat function to our agents file.

File: `convex/agents.ts`

Add this to the existing file:

// Mutation: Record agent heartbeat
export const heartbeat = mutation({
  args: {
    name: v.string(),
    status: v.optional(
      v.union(v.literal("online"), v.literal("offline"), v.literal("error"))
    ),
    metadata: v.optional(
      v.object({
        version: v.optional(v.string()),
        platform: v.optional(v.string()),
        tags: v.optional(v.array(v.string())),
      })
    ),
  },
  handler: async (ctx, args) => {
    // Find agent by name
    const existingAgent = await ctx.db
      .query("agents")
      .withIndex("by_name", (q) => q.eq("name", args.name))
      .unique();

    const now = Date.now();

    if (existingAgent) {
      // Update existing agent
      await ctx.db.patch(existingAgent._id, {
        status: args.status ?? "online",
        lastHeartbeat: now,
        metadata: args.metadata ?? existingAgent.metadata,
      });

      return {
        agentId: existingAgent._id,
        action: "updated",
      };
    } else {
      // Create new agent
      const agentId = await ctx.db.insert("agents", {
        name: args.name,
        status: args.status ?? "online",
        lastHeartbeat: now,
        metadata: args.metadata,
      });

      return {
        agentId,
        action: "created",
      };
    }
  },
});
Understanding the Heartbeat Mutation
Find existing agent:

const existingAgent = await ctx.db
  .query("agents")
  .withIndex("by_name", (q) => q.eq("name", args.name))
  .unique();
Uses the by_name index for fast lookup

.unique() returns one result (or null)

Auto-create or update:

if (existingAgent) {
  // Update existing
  await ctx.db.patch(existingAgent._id, { ... });
} else {
  // Create new
  await ctx.db.insert("agents", { ... });
}
This pattern lets agents self-register. No manual setup needed!

Return feedback:

return {
  agentId: existingAgent._id,
  action: "updated", // or "created"
};
Tells the agent whether it was created or updated.

import { httpRouter } from "convex/server";
import { httpAction } from "./_generated/server";
import { api } from "./_generated/api";
import { v } from "convex/values";

const http = httpRouter();

// POST /api/heartbeat
http.route({
  path: "/api/heartbeat",
  method: "POST",
  handler: httpAction(async (ctx, request) => {
    try {
      // Parse request body
      const body = await request.json();
      
      // Validate required fields
      if (!body.name || typeof body.name !== "string") {
        return new Response(
          JSON.stringify({ error: "Agent name is required" }),
          {
            status: 400,
            headers: { "Content-Type": "application/json" },
          }
        );
      }

      // Call the heartbeat mutation
      const result = await ctx.runMutation(api.agents.heartbeat, {
        name: body.name,
        status: body.status,
        metadata: body.metadata,
      });

      return new Response(JSON.stringify(result), {
        status: 200,
        headers: { "Content-Type": "application/json" },
      });
    } catch (error: any) {
      console.error("Heartbeat error:", error);
      return new Response(
        JSON.stringify({ error: error.message }),
        {
          status: 500,
          headers: { "Content-Type": "application/json" },
        }
      );
    }
  }),
});

export default http;

Then add at the bottom:

// Internal mutation: Check for timed-out agents
export const checkTimeouts = internalMutation({
  handler: async (ctx) => {
    const now = Date.now();
    const timeoutMs = 2 * 60 * 1000; // 2 minutes
    const cutoff = now - timeoutMs;

    // Find agents that haven't sent heartbeats recently
    const allAgents = await ctx.db.query("agents").collect();
    
    let updated = 0;
    for (const agent of allAgents) {
      // Skip if already offline/error
      if (agent.status !== "online") continue;
      
      // Check if heartbeat is stale
      if (agent.lastHeartbeat < cutoff) {
        await ctx.db.patch(agent._id, {
          status: "offline",
        });
        
        // Log activity
        await ctx.db.insert("activities", {
          agentId: agent._id,
          type: "warning",
          message: "Agent heartbeat timeout - marked offline",
          timestamp: now,
        });
        
        updated++;
      }
    }

    return { checked: allAgents.length, updated };
  },
});
Understanding the Timeout Checker
Run every minute:

crons.interval("check agent timeouts", { minutes: 1 }, ...);
Cron job runs automatically every 60 seconds.

Define timeout threshold:

const timeoutMs = 2 * 60 * 1000; // 2 minutes
const cutoff = now - timeoutMs;
If last heartbeat is older than 2 minutes, agent is considered dead.

Find stale agents:

for (const agent of allAgents) {
  if (agent.status !== "online") continue;
  if (agent.lastHeartbeat < cutoff) {
    // Mark as offline
  }
}
Log the timeout:

await ctx.db.insert("activities", {
  agentId: agent._id,
  type: "warning",
  message: "Agent heartbeat timeout - marked offline",
  timestamp: now,
});
Creates an activity record so you can see when timeouts happened.

Why `internalMutation`?

Internal mutations can only be called by cron jobs or other internal functions. This prevents external callers from triggering timeout checks.

"use client";

export function AgentStatusIndicator({
  status,
  lastHeartbeat,
  size = "md",

      <div
        className={`rounded-full ${sizes[size]} ${styles[status]} shadow-lg`}
      />
      <span className="text-sm text-gray-400">{labels[status]}</span>

  );

"use client";

import { useState } from "react";
import { useMutation, useQuery } from "convex/react";
import { api } from "../../convex/_generated/api";
import { Id } from "../../convex/_generated/dataModel";

export function TaskForm({ onClose }: { onClose: () => void }) {
  const [title, setTitle] = useState("");
  const [description, setDescription] = useState("");
  const [priority, setPriority] = useState<"low" | "medium" | "high">("medium");
  const [assignedTo, setAssignedTo] = useState<Id<"agents"> | "">("");
  const [dueInHours, setDueInHours] = useState("");

  const createTask = useMutation(api.tasks.createTask);
  const agents = useQuery(api.agents.getAllAgents);

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();

    if (!title.trim()) {
      alert("Task title is required");
      return;
    }

    const dueAt = dueInHours
      ? Date.now() + parseInt(dueInHours) * 60 * 60 * 1000
      : undefined;

    try {
      await createTask({
        title: title.trim(),
        description: description.trim() || undefined,
        priority,
        assignedTo: assignedTo || undefined,
        dueAt,
      });

      // Reset form
      setTitle("");
      setDescription("");
      setPriority("medium");
      setAssignedTo("");
      setDueInHours("");
      
      onClose();
    } catch (error) {
      console.error("Failed to create task:", error);
      alert("Failed to create task. Please try again.");
    }
  };

  return (
    <form onSubmit={handleSubmit} className="space-y-4">
      <div>
        <label htmlFor="title" className="block text-sm font-medium mb-1">
          Task Title *
        </label>
        <input
          id="title"
          type="text"
          value={title}
          onChange={(e) => setTitle(e.target.value)}
          className="w-full px-3 py-2 bg-gray-800 border border-gray-700 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
          placeholder="e.g., Process customer emails"
          required
        />
      </div>

      <div>
        <label htmlFor="description" className="block text-sm font-medium mb-1">
          Description
        </label>
        <textarea
          id="description"
          value={description}
          onChange={(e) => setDescription(e.target.value)}
          className="w-full px-3 py-2 bg-gray-800 border border-gray-700 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
          placeholder="Optional details about the task"
          rows={3}
        />
      </div>

      <div className="grid grid-cols-2 gap-4">
        <div>
          <label htmlFor="priority" className="block text-sm font-medium mb-1">
            Priority
          </label>
          <select
            id="priority"
            value={priority}
            onChange={(e) => setPriority(e.target.value as any)}
            className="w-full px-3 py-2 bg-gray-800 border border-gray-700 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
          >
            <option value="low">Low</option>
            <option value="medium">Medium</option>
            <option value="high">High</option>
          </select>
        </div>

        <div>
          <label htmlFor="assignedTo" className="block text-sm font-medium mb-1">
            Assign To
          </label>
          <select
            id="assignedTo"
            value={assignedTo}
            onChange={(e) => setAssignedTo(e.target.value as any)}
            className="w-full px-3 py-2 bg-gray-800 border border-gray-700 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
          >
            <option value="">Unassigned</option>
            {agents?.map((agent) => (
              <option key={agent._id} value={agent._id}>
                {agent.name}
              </option>
            ))}
          </select>
        </div>
      </div>

      <div>
        <label htmlFor="dueInHours" className="block text-sm font-medium mb-1">

"use client";

import { useMutation } from "convex/react";
import { api } from "../../convex/_generated/api";
import { Id } from "../../convex/_generated/dataModel";

type Task = {
  _id: Id<"tasks">;
  title: string;
  description?: string;
  status: "pending" | "in-progress" | "complete" | "failed";
  priority: "low" | "medium" | "high";
  assignedTo?: Id<"agents">;
  agentName?: string | null;
  createdAt: number;
  dueAt?: number;
};

export function TaskCard({ task }: { task: Task }) {
  const updateStatus = useMutation(api.tasks.updateTaskStatus);

  const priorityColors = {
    low: "border-blue-500/30 bg-blue-500/5",
    medium: "border-yellow-500/30 bg-yellow-500/5",
    high: "border-red-500/30 bg-red-500/5",
  };

  const priorityBadges = {
    low: "bg-blue-500/10 text-blue-400 border-blue-500/20",
    medium: "bg-yellow-500/10 text-yellow-400 border-yellow-500/20",
    high: "bg-red-500/10 text-red-400 border-red-500/20",
  };

  const statusActions = {
    pending: { label: "Start", nextStatus: "in-progress" as const },
    "in-progress": { label: "Complete", nextStatus: "complete" as const },
    complete: { label: "Completed", nextStatus: null },
    failed: { label: "Failed", nextStatus: null },
  };

  const action = statusActions[task.status];

  const handleStatusChange = async () => {
    if (!action.nextStatus) return;
    
    try {
      await updateStatus({
        taskId: task._id,
        status: action.nextStatus,
      });
    } catch (error) {
      console.error("Failed to update task:", error);
    }
  };

  const isOverdue = task.dueAt && task.dueAt < Date.now() && task.status !== "complete";

  return (
    <div
      className={`p-4 rounded-lg border ${priorityColors[task.priority]} hover:border-gray-600 transition-colors`}
    >
      <div className="flex items-start justify-between mb-2">
        <h3 className="font-semibold text-white">{task.title}</h3>
        <span
          className={`px-2 py-1 rounded text-xs font-semibold border ${priorityBadges[task.priority]}`}
        >
          {task.priority}
        </span>
      </div>

      {task.description && (
        <p className="text-sm text-gray-400 mb-3">{task.description}</p>
      )}

      <div className="space-y-2 text-xs text-gray-500">
        {task.agentName && (
          <div className="flex items-center gap-1">
            <svg className="w-4 h-4" fill="none" stroke="currentColor" viewBox="0 0 24 24">
              <path strokeLinecap="round" strokeLinejoin="round" strokeWidth={2} d="M16 7a4 4 0 11-8 0 4 4 0 018 0zM12 14a7 7 0 00-7 7h14a7 7 0 00-7-7z" />
            </svg>
            <span>{task.agentName}</span>
          </div>
        )}

        {task.dueAt && (
          <div className={`flex items-center gap-1 ${isOverdue ? "text-red-400" : ""}`}>
            <svg className="w-4 h-4" fill="none" stroke="currentColor" viewBox="0 0 24 24">
              <path strokeLinecap="round" strokeLinejoin="round" strokeWidth={2} d="M12 8v4l3 3m6-3a9 9 0 11-18 0 9 9 0 0118 0z" />
            </svg>
            <span>
              {isOverdue ? "Overdue: " : "Due: "}
              {formatTimestamp(task.dueAt)}
            </span>
          </div>
        )}
      </div>

      {action.nextStatus && (
        <button
          onClick={handleStatusChange}
          className="w-full mt-3 px-3 py-2 bg-gray-700 hover:bg-gray-600 rounded text-sm font-medium transition-colors"
        >
          {action.label}
        </button>
      )}
    </div>
  );
}

function formatTimestamp(timestamp: number): string {
  const date = new Date(timestamp);
  const now = new Date();
  const diffMs = timestamp - now.getTime();
  const diffHours = Math.floor(diffMs / (1000 * 60 * 60));
  
  if (diffHours < 0) return `${Math.abs(diffHours)}h ago`;
  if (diffHours < 24) return `in ${diffHours}h`;
  return date.toLocaleDateString();
}

"use client";

import { useState } from "react";
import { useQuery } from "convex/react";
import { api } from "../../convex/_generated/api";
import { Id } from "../../convex/_generated/dataModel";

export function ActivityFeed() {
  const [filterType, setFilterType] = useState<string>("all");
  const [filterAgent, setFilterAgent] = useState<Id<"agents"> | "all">("all");
  const [limit, setLimit] = useState(50);

  const activities = useQuery(api.activities.getAllActivities, { limit });
  const agents = useQuery(api.agents.getAllAgents);

  if (activities === undefined || agents === undefined) {
    return (
      <div className="p-8 text-gray-400">
        Loading activities...
      </div>
    );
  }

  // Apply filters
  let filteredActivities = activities;
  
  if (filterType !== "all") {
    filteredActivities = filteredActivities.filter(
      (a) => a.type === filterType
    );
  }

function formatTimestamp(timestamp: number): string {
  const seconds = Math.floor((Date.now() - timestamp) / 1000);
  
  if (seconds < 10) return "just now";
  if (seconds < 60) return `${seconds}s ago`;
  if (seconds < 3600) return `${Math.floor(seconds / 60)}m ago`;
  if (seconds < 86400) return `${Math.floor(seconds / 3600)}h ago`;
  return `${Math.floor(seconds / 86400)}d ago`;
}

Step 3: Add Activity Logging to Heartbeat
Let's automatically log heartbeat activities.

File: `convex/agents.ts`

Update the heartbeat mutation:

export const heartbeat = mutation({
  args: {
    name: v.string(),
    status: v.optional(
      v.union(v.literal("online"), v.literal("offline"), v.literal("error"))
    ),
    metadata: v.optional(
      v.object({
        version: v.optional(v.string()),
        platform: v.optional(v.string()),
        tags: v.optional(v.array(v.string())),
      })
    ),
  },
  handler: async (ctx, args) => {
    const existingAgent = await ctx.db
      .query("agents")
      .withIndex("by_name", (q) => q.eq("name", args.name))
      .unique();

    const now = Date.now();
    const newStatus = args.status ?? "online";

    if (existingAgent) {
      // Check if status changed
      const statusChanged = existingAgent.status !== newStatus;

      // Update existing agent
      await ctx.db.patch(existingAgent._id, {
        status: newStatus,
        lastHeartbeat: now,
        metadata: args.metadata ?? existingAgent.metadata,
      });

      // Log activity if status changed
      if (statusChanged) {
        await ctx.db.insert("activities", {
          agentId: existingAgent._id,
          type: newStatus === "online" ? "success" : newStatus === "error" ? "error" : "info",
          message: `Status changed to ${newStatus}`,
          timestamp: now,
        });
      }

      return {
        agentId: existingAgent._id,
        action: "updated",
      };
    } else {
      // Create new agent
      const agentId = await ctx.db.insert("agents", {
        name: args.name,
        status: newStatus,
        lastHeartbeat: now,
        metadata: args.metadata,
      });

      // Log creation activity
      await ctx.db.insert("activities", {
        agentId,
        type: "info",
        message: "Agent registered",
        timestamp: now,
      });

      return {
        agentId,
        action: "created",
      };
    }
  },
});

export const updateTaskStatus = mutation({
  args: {
    taskId: v.id("tasks"),
    status: v.union(
      v.literal("pending"),
      v.literal("in-progress"),
      v.literal("complete"),
      v.literal("failed")
    ),
  },
  handler: async (ctx, args) => {
    const task = await ctx.db.get(args.taskId);
    if (!task) throw new Error("Task not found");

    const now = Date.now();
    const updates: any = {
      status: args.status,
      updatedAt: now,
    };

    if (args.status === "complete" || args.status === "failed") {
      updates.completedAt = now;
    }

    await ctx.db.patch(args.taskId, updates);

    // Log activity if task is assigned to an agent
    if (task.assignedTo) {
      const activityType = 
        args.status === "complete" ? "success" :
        args.status === "failed" ? "error" :
        "info";

      await ctx.db.insert("activities", {
        agentId: task.assignedTo,
        type: activityType,
        message: `Task "${task.title}" marked as ${args.status}`,
        timestamp: now,
      });
    }
  },
});

function formatTimestamp(timestamp: number): string {
  const seconds = Math.floor((Date.now() - timestamp) / 1000);
  
  if (seconds < 60) return `${seconds}s ago`;
  if (seconds < 3600) return `${Math.floor(seconds / 60)}m ago`;
  if (seconds < 86400) return `${Math.floor(seconds / 3600)}h ago`;
  return new Date(timestamp).toLocaleString();
}

Understanding Real-Time in Convex
How Subscriptions Work
When you use useQuery:

const agents = useQuery(api.agents.getAllAgents);
What happens:

Initial fetch: Convex fetches data immediately

Subscribe: Opens a WebSocket connection

Listen: Server pushes updates when data changes

Re-render: React automatically updates your UI

No polling. No manual refreshing. Just real-time data.

The Magic Behind It
Traditional approach (polling):

Every 5 seconds:
  fetch("/api/agents")
  update UI
Problems:

❌ Wastes bandwidth

❌ Delayed updates (5 second lag)

❌ Battery drain

❌ Complex code

Convex approach (push):

Subscribe once:
  receive updates instantly when data changes
Benefits:

✅ Instant updates (milliseconds)

✅ Minimal bandwidth

✅ Battery efficient

✅ Simple code

"use client";

import { useEffect, useState } from "react";

export function ConnectionStatus() {
  const [isOnline, setIsOnline] = useState(true);
  const [showStatus, setShowStatus] = useState(false);

  useEffect(() => {
    const handleOnline = () => {
      setIsOnline(true);
      setShowStatus(true);
      setTimeout(() => setShowStatus(false), 3000);
    };

    const handleOffline = () => {
      setIsOnline(false);
      setShowStatus(true);
    };

    window.addEventListener("online", handleOnline);
    window.addEventListener("offline", handleOffline);

    return () => {
      window.removeEventListener("online", handleOnline);
      window.removeEventListener("offline", handleOffline);
    };
  }, []);

  if (!showStatus && isOnline) return null;

  return (
    <div className="fixed bottom-4 right-4 z-50">
      <div
        className={`px-4 py-3 rounded-lg shadow-lg border ${
          isOnline
            ? "bg-green-500/10 border-green-500/30 text-green-400"
            : "bg-red-500/10 border-red-500/30 text-red-400"
        } animate-in slide-in-from-bottom-5`}
      >
        <div className="flex items-center gap-3">
          <div
            className={`w-2 h-2 rounded-full ${
              isOnline ? "bg-green-500 animate-pulse" : "bg-red-500"
            }`}
          />
          <span className="font-medium">
            {isOnline ? "Connected" : "Connection lost"}
          </span>
        </div>
      </div>
    </div>
  );
}

Step 3: Implement Optimistic UI for Tasks
Optimistic UI updates the interface immediately, before the server confirms. This makes the app feel instant.

"use client";

import { useState } from "react";
import { useMutation } from "convex/react";
import { api } from "../../convex/_generated/api";
import { Id } from "../../convex/_generated/dataModel";

type Task = {
  _id: Id<"tasks">;
  title: string;
  description?: string;
  status: "pending" | "in-progress" | "complete" | "failed";
  priority: "low" | "medium" | "high";
  assignedTo?: Id<"agents">;
  agentName?: string | null;
  createdAt: number;
  dueAt?: number;
};

export function TaskCard({ task }: { task: Task }) {
  const updateStatus = useMutation(api.tasks.updateTaskStatus);
  const [isUpdating, setIsUpdating] = useState(false);
  const [optimisticStatus, setOptimisticStatus] = useState(task.status);

  const priorityColors = {
    low: "border-blue-500/30 bg-blue-500/5",
    medium: "border-yellow-500/30 bg-yellow-500/5",
    high: "border-red-500/30 bg-red-500/5",
  };

  const priorityBadges = {
    low: "bg-blue-500/10 text-blue-400 border-blue-500/20",
    medium: "bg-yellow-500/10 text-yellow-400 border-yellow-500/20",
    high: "bg-red-500/10 text-red-400 border-red-500/20",
  };

  const statusActions = {
    pending: { label: "Start", nextStatus: "in-progress" as const },
    "in-progress": { label: "Complete", nextStatus: "complete" as const },
    complete: { label: "Completed", nextStatus: null },
    failed: { label: "Failed", nextStatus: null },
  };

  // Use optimistic status if updating, otherwise use real status
  const displayStatus = isUpdating ? optimisticStatus : task.status;
  const action = statusActions[displayStatus];

  const handleStatusChange = async () => {
    if (!action.nextStatus) return;
    
    // Optimistic update
    setIsUpdating(true);
    setOptimisticStatus(action.nextStatus);
    
    try {
      await updateStatus({
        taskId: task._id,
        status: action.nextStatus,
      });
    } catch (error) {
      console.error("Failed to update task:", error);
      // Revert on error
      setOptimisticStatus(task.status);
    } finally {
      setIsUpdating(false);
    }
  };

  const isOverdue =
    task.dueAt && task.dueAt < Date.now() && displayStatus !== "complete";

  return (
    <div
      className={`p-4 rounded-lg border ${priorityColors[task.priority]} hover:border-gray-600 transition-colors ${
        isUpdating ? "opacity-70" : ""
      }`}
    >
      <div className="flex items-start justify-between mb-2">
        <h3 className="font-semibold text-white">{task.title}</h3>
        <span
          className={`px-2 py-1 rounded text-xs font-semibold border ${priorityBadges[task.priority]}`}
        >
          {task.priority}
        </span>
      </div>

      {task.description && (
        <p className="text-sm text-gray-400 mb-3">{task.description}</p>
      )}

      <div className="space-y-2 text-xs text-gray-500">
        {task.agentName && (
          <div className="flex items-center gap-1">
            <svg
              className="w-4 h-4"
              fill="none"
              stroke="currentColor"
              viewBox="0 0 24 24"
            >
              <path
                strokeLinecap="round"
                strokeLinejoin="round"
                strokeWidth={2}
                d="M16 7a4 4 0 11-8 0 4 4 0 018 0zM12 14a7 7 0 00-7 7h14a7 7 0 00-7-7z"
              />
            </svg>
            <span>{task.agentName}</span>
          </div>
        )}

        {task.dueAt && (
          <div
            className={`flex items-center gap-1 ${isOverdue ? "text-red-400" : ""}`}
          >
            <svg
              className="w-4 h-4"
              fill="none"
              stroke="currentColor"
              viewBox="0 0 24 24"
            >
              <path
                strokeLinecap="round"
                strokeLinejoin="round"
                strokeWidth={2}
                d="M12 8v4l3 3m6-3a9 9 0 11-18 0 9 9 0 0118 0z"
              />
            </svg>
            <span>
              {isOverdue ? "Overdue: " : "Due: "}
              {formatTimestamp(task.dueAt)}
            </span>
          </div>
        )}
      </div>

      {action.nextStatus && (
        <button
          onClick={handleStatusChange}
          disabled={isUpdating}
          className="w-full mt-3 px-3 py-2 bg-gray-700 hover:bg-gray-600 rounded text-sm font-medium transition-colors disabled:opacity-50 disabled:cursor-not-allowed"
        >
          {isUpdating ? "Updating..." : action.label}
        </button>
      )}
    </div>
  );
}

function formatTimestamp(timestamp: number): string {
  const date = new Date(timestamp);
  const now = new Date();
  const diffMs = timestamp - now.getTime();
  const diffHours = Math.floor(diffMs / (1000 * 60 * 60));
  
  if (diffHours < 0) return `${Math.abs(diffHours)}h ago`;
  if (diffHours < 24) return `in ${diffHours}h`;
  return date.toLocaleDateString();
}

Step 5: Optimize Query Performance
Limit Data Fetching
Instead of fetching all data, limit queries:

// ❌ Bad: Fetches everything
const activities = useQuery(api.activities.getAllActivities);

// ✅ Good: Limits to 50
const activities = useQuery(api.activities.getAllActivities, { limit: 50 });
Use Specific Queries
// ❌ Bad: Fetches all agents, filters client-side
const agents = useQuery(api.agents.getAllAgents);
const onlineAgents = agents?.filter(a => a.status === "online");

// ✅ Good: Filters on server
const onlineAgents = useQuery(api.agents.getAgentsByStatus, { status: "online" });

export const getActivitiesPaginated = query({
  args: {
    limit: v.number(),
    cursor: v.optional(v.string()),
  },
  handler: async (ctx, args) => {
    const results = await ctx.db
      .query("activities")
      .withIndex("by_timestamp")
      .order("desc")
      .paginate({
        numItems: args.limit,
        cursor: args.cursor ? args.cursor : null,
      });

    return results;
  },
});

"use client";

import { useEffect, useState } from "react";

export function DataFreshness({ timestamp }: { timestamp?: number }) {
  const [, setNow] = useState(Date.now());

  useEffect(() => {
    const interval = setInterval(() => {
      setNow(Date.now());
    }, 1000);

    return () => clearInterval(interval);
  }, []);

  if (!timestamp) return null;

  const seconds = Math.floor((Date.now() - timestamp) / 1000);
  
  let color = "text-green-400";
  let label = "Fresh";
  
  if (seconds > 60) {
    color = "text-yellow-400";
    label = "Stale";
  }
  
  if (seconds > 300) {
    color = "text-red-400";
    label = "Very stale";
  }

  return (
    <div className="flex items-center gap-2 text-xs">
      <div className={`w-2 h-2 rounded-full ${color.replace("text-", "bg-")}`} />
      <span className={color}>
        {label} ({seconds}s ago)
      </span>
    </div>
  );
}
